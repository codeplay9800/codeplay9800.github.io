Bug Shooter

About

Bug Shooter initially started as a gradable assignment for a FutureGames course. Afterwards, I kept working on the project, adding a second spatial data structure to compare the results and a background generator by using the Wave Function Collapse algorithm.

Project Info

Role: Gameplay Programmer

Team Size: 1

Time frame: 6 Weeks

Engine: SDL 2 (C++)

Introduction

This space shooter game was a fun and challenging project for me, where I had the opportunity to work on creating my own ECS in C++, focusing on optimization and getting a better understanding of the Performance Profiler in Visual Studio.

In order to gain a better understanding of procedural generation, I wanted to build a background generator using the wave function collapse algorithm. I decided on the wave function collapse algorithm because it appealed the most to me, and it can be used in a variety of ways.

Wave Function Collapse (WFC)

I want to clarify that my version of the wave function collapse algorithm does not use a reference image or perform overlap checking to generate unique tiles and assign weights based on their frequency in the image. Instead, I started small and created a simple implementation that uses a set of predefined rules to determine which tiles can be placed next to each other.

Spatial Hash vs Quadtree

My game was already running smooth, but I wanted to see if I could do even better by switching to another spatial data structure. I decided to additionally add a quadtree to compare the frames between it and the spatial hash that I was already using.

In my very simple unscientific tests, I found that the game had more fps when using the quadtree with 35k moving entities, but that the spatial hash was slightly better when the entities stood still. This made me believe that the cause has to do with the remove/insert implementation difference between the two structures.

I believe the performance difference between the spatial hash and quadtree structures in my game is due to the complexity of their insert and remove functions. The spatial hash has O(n^2) complexity for both insert and remove, while the quadtree has O(log n) complexity for both. These functions are called frequently when entities update their positions, so their complexity can significantly impact the game's performance. I believe that either structure could be used in a game, depending on the specific needs and requirements of the project. It's even possible to use both structures in the same game, depending on the circumstances.
