Overview

This project compares the performance of Forward, Forward+ clustered, and Deferred clustered shading schemes.

Shading Schemes

• Forward: calculates the visible object mesh once and iterates through all lights in the scene.
• Forward+: divides the scene into tiled frustum clusters and shades objects only with affecting lights.
• Deferred Clustered: uses multiple passes and stores scene information in G-Buffers.

Performance Analysis

In theory, the expected performance order is: Deferred Clustered > Forward+ > Forward.

Analysis was done using Google Chrome's built-in performance tools.
